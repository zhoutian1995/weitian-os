01 Step One

买域名

5分钟搞定 ✓

Cloudflare 里选名字

域名就是你在互联网上的"门牌号"。 买域名 = 把这个名字注册成你的，别人就不能用了。 这是拥有自己网站的必经之路。

我用的是 Cloudflare——没有溢价，价格透明，界面也不复杂。

1 去 cloudflare.com 注册账号
2 左侧菜单「域名注册」→ 搜索你想要的名字
3 选一个你喜欢的后缀：
.me .design .dev .ai .xyz
（我选的 .me，很适合个人品牌）
4 付款，搞定。域名就是你的了。
💡 小技巧

选名字的时候让 AI 帮你头脑风暴——给它你的 ID、你的名字、你想要的风格， 它会给你列一堆方案。我买了两年的，一年100多块，比想象中便宜很多。

Step Two

让域名指向你的网页

← 把门牌号挂到门口 ✓
← 就是这个！两边各配置一下

这一步在做什么？简单说：你买了个门牌号，但门牌号还没有挂到你家门口。 现在要做的就是把门牌号挂上去——让别人输入 hiesther.me 的时候，能找到你的网页存放的地方。

需要在两个地方各操作一下 👇

2.1 · GitHub

告诉 GitHub："我有域名了"

这一步让 GitHub 知道：以后通过 hiesther.me 来访问时，把我这个仓库的网页展示给他。

1 仓库 → Settings → Pages
2 Custom domain 栏输入你的域名，点 Save
3 等检查通过后，勾选 Enforce HTTPS

GitHub 会自动在仓库里生成一个 CNAME 文件，你不用管。

2.2 · Cloudflare

告诉互联网："hiesther.me 在这里"

电脑不认识名字，只认识数字地址（IP）。你需要告诉全世界："hiesther.me 请去这里找"。

登录 Cloudflare → 点域名 → DNS → 记录 → 添加4条 A 记录：

类型 名称 内容
A @ 185.199.108.153
A @ 185.199.109.153
A @ 185.199.110.153
A @ 185.199.111.153
⚠️

代理状态一定要关掉（灰色云，不是橙色云）。GitHub Pages 不支持 Cloudflare 代理。

← 快的话几分钟，慢的话几小时。等就行了。

DNS 生效需要一点时间——快的话几分钟，慢的话几小时。 这是全球互联网系统在同步你的新地址，不是你操作出了问题。

等它就行了。 期间可以刷新 Cloudflare 的 DNS 记录页，看有没有报错。 一般来说，只要记录填对了，就没问题。

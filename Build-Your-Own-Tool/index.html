01 - Agent

你的技术搭档

沟通需求

先跟Cola聊清楚到底要什么--不是直接说"帮我做个App",而是一起讨论:管哪些东西?谁会用?什么场景下用?

让它写PRD

需求聊明白后,让它整理成产品文档--功能列表、数据结构、页面框架。确认"我们理解的是同一件事"。

小步迭代

不是一口气写完。先做最小可用版本,跑通了再加下一个功能。每次加一点,每次确认方向对不对。

你的角色:提需求、验收结果、说"不对"

Agent的角色:写代码、给选择、引导技术决策

02~04 - The Tools

每个工具解决一个"为什么"

☁️GitHub
🗄️Supabase
🚀Vercel

代码的云端存档

为什么不只存电脑上?因为GitHub可以回滚--每次更新都有记录,改坏了可以退回上一个版本。当你的项目越来越大,这个追溯历史版本的能力非常重要。

而且后面Vercel部署时,直接从GitHub拿你的代码。它们之间是打通的。

你的免费后端

你跟Agent说"我需要多人能注册,要能存图片"--它会引导你用Supabase来解决:数据库 + 文件存储 + 用户系统 + 实时同步。

免费额度对个人项目完全够用,不需要花钱。

部署上线

你跟Agent说"我想让别人也能通过网址访问"--它引导你用Vercel。绑定GitHub仓库,自动部署,给你一个网址。不用额外买域名。

以后每次更新代码推到GitHub,Vercel自动重新上线。改了就生效,零操作。
